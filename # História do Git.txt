# História do Git

O **Git** é um sistema de controle de versão distribuído criado por **Linus Torvalds** em 2005. Ele surgiu da necessidade de uma ferramenta eficiente para gerenciar o desenvolvimento do kernel do Linux, após a empresa BitKeeper deixar de oferecer sua solução gratuitamente para a comunidade.

## Principais Marcos

- **2005**: Linus Torvalds inicia o desenvolvimento do Git, focando em velocidade, integridade dos dados e suporte ao trabalho distribuído.
- **Primeira versão**: Em poucas semanas, o Git já estava funcional e começou a ser utilizado no desenvolvimento do kernel Linux.
- **Desenvolvimento contínuo**: Desde então, o Git evoluiu rapidamente, recebendo contribuições de desenvolvedores do mundo todo.

## Características

- **Distribuído**: Cada desenvolvedor possui uma cópia completa do repositório.
- **Rápido**: Operações como commit, branch e merge são otimizadas.
- **Seguro**: Utiliza algoritmos de hash (SHA-1) para garantir a integridade dos dados.
- **Popularidade**: Tornou-se o sistema de controle de versão mais utilizado no mundo, sendo a base de plataformas como GitHub, GitLab e Bitbucket.

## Impacto

O Git revolucionou a forma como projetos de software são desenvolvidos colaborativamente, facilitando o trabalho em equipe, o gerenciamento de versões e a integração contínua.
